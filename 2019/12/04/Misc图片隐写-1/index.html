Misc图片隐写-1

发表于 2019-12-04 | 分类于 CTF

之前刷的一些杂项的图片隐写题，大多为Buuctf上的题目。

图片隐写类

1、buuctf misc2 二维码

给了一个二维码图片 //分割图片
binwalk -e QR_code.png

发现图片中包含了一个加密的zip文件和一个空的txt文件，txt文件名为4number，暗示zip密码为4位数字。
fcrackzip -b -c1 -l 4 -u target.zip
注：
-b 使用暴力破解的方式
-c 1 使用数字的组合
-c ‘aA1’ 使用大小写字母和数字的组合
-l 4 需要破解的密码长度为4位
-l 1-10 需要破解的密码长度为1-10位
-u 只显示破解出来的密码，其他错误的不显示
-D -p password.txt 使用这个字典进行破解

打开zip获取到flag：CTF{vjpw_wnoei}

2、buuctf misc3 金三胖.gif

给了一个gif动图，直接查看动图发现中间有几张图片一闪而过。
使用 stegsolve
注：
File Format:文件格式，这个主要是查看图片的具体信息
Data Extract:数据抽取，图片中隐藏数据的抽取
Frame Browser:帧浏览器，主要是对GIF之类的动图进行分解，动图变成一张张图片，便于查看
Image Combiner:拼图，图片拼接
本题直接使用Frame Browser一帧一帧看即可获取到flag

flag{he11ohongke}

这里可以了解一下PNG文件头知识：

（固定）八个字节89 50 4E 47 0D 0A 1A 0A为png的文件头 （固定）四个字节00 00 00 0D（即为十进制的13）代表数据块的长度为13 （固定）四个字节49 48 44 52（即为ASCII码的IHDR）是文件头数据块的标示（IDCH） （可变）13位数据块（IHDR) --- 前四个字节代表该图片的宽 --- 后四个字节代表该图片的高 --- 后五个字节依次为： Bit depth、ColorType、Compression method、Filter method、Interlace method （可变）剩余四字节为该png的CRC检验码，由从IDCH到IHDR的十七位字节进行crc计算得到。

使用脚本计算正确的高度

5、buuctf misc8 LSB

LSB图片隐写
前置知识：

LSB图片隐写：png图片是一种无损压缩的位图片形格式，也只有在无损压缩或者无压缩的图片（BMP）上实现lsb隐写。如果图像是jpg图片的话，就没法使用lsb隐写了，原因是jpg图片对像数进行了有损压缩，我们修改的信息就可能会在压缩的过程中被破坏。而png图片虽然也有压缩，但却是无损压缩，这样我们修改的信息也就能得到正确的表达，不至于丢失。BMP图片一般是特别的大的，因为BMP把所有的像数都按原样储存，没有进行压缩。 png图片中的图像像数一般是由RGB三原色（红绿蓝）组成，每一种颜色占用8位，取值范围为0x00~0xFF（0-255），即有256种颜色，一共包含了256的3次方的颜色，即16777216种颜色。而人类的眼睛可以区分约1000万种不同的颜色，这就意味着人类的眼睛无法区分余下的颜色大约有6777216种。 LSB隐写就是修改RGB颜色分量的最低二进制位也就是最低有效位（LSB），而人类的眼睛不会注意到这前后的变化，每个像数可以携带3比特的信息。 一幅完整的图像，红色绿色蓝色三个通道缺一不可。一幅图像，如果关闭了红色通道，那么图像就偏青色。如果关闭了绿色通道，那么图像就偏洋红色。如果关闭了蓝色通道，那么图像就偏黄色。当然还有个Alpha通道，是一个8位的灰度通道，也可以理解为透明度（粗糙的理解）。

关键点：
LSB隐写就是修改RGB颜色分量的最低二进制位也就是最低有效位（LSB）
使用stegsolve打开图片，一直向右看各通道显示的图像，可以看到Red、Blue、Green的0通道的图片上方有东西：



用Data Extract查看这三个通道结合起来的东西，Save Bin将图片保存为png图片：

发现是一个二维码，扫一扫即可得到flag。

010打开看到有个flag.txt

14、buuctf misc 18 假如给我三天光明

得到一张图片和一个zip
图片下面有一串盲文，使用盲文对照表，得到：KMDONOWG

小写作为密码打开zip得到音频文件,打开发现是摩斯密码
使用Audacity打开摩斯密码
其中宽的为“-”，短的为“.”，中间用空格或/分隔，但不能同时使用两种

注：这里的null为？，对应的莫斯密码为 ..–..
去掉ctf，把所有字符小写，即为flag

15、buuctf misc 20 FLAG

png binwalk分离没东西
查看LSB，发现文件头为50 4B 03 04 ,是一个zip文件

save bin 保存为zip文件，打开发现了一个文件1。
（1）使用16进制编辑器打开，发现为elf文件，其中看到了flag。
（2）使用file命令查看文件类型，使用strings也可以拿到flag。
