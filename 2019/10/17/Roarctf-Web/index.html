Web-1 easy_calc

F12（目标存在WAF）

被WAF拦截

此处WAF的部署方式应该是反向代理的部署方式。
绕过WAF需要利用请求走私漏洞（参考：https://paper.seebug.org/1048/）

这里可以使用burp插件进行测试

测试完的结果会放在插件Flow中.

这里不算事请求走私的典型场景（如TE-CL、CL-TE等）。

这里代理服务器（WAF）返回的是400，请求有误。是因为它发现请求头中既有CL也有TE，它认为这个请求是错误的。

但是它这里把这个错误的请求发送到了后端服务器，后端服务器认为这个请求正常，直接就响应了，这样就绕过了中间的WAF。

代码执行

有了phpinfo之后需要关注一下disable_functions，可以看到exec、shell_exec、system等函数都被禁用了。

找flag我们需要先列目录。从phpinfo里可以看到chdir已被禁用，这里用到了scandir()
直接使用scandir返回的是数组，还需要用var_dump()输出数组的详细信息。
var_dump(scandir('/'));

但是这里还存在对单引号、斜杠等符号的过滤，使用几个进制转换的函数绕过。

base_convert() 函数在任意进制之间转换数字。
例：base_convert(55490343972,10,36)() == phpinfo()

dechex() 函数把十进制数转换为十六进制数。

hex2bin() 函数把十六进制值的字符串转换为 ASCII 字符。

构造payload

var_dump(scandir(/))
var_dump(base_convert(61693386291,10,36)(hex2bin(dechex(47))))

注：scandir这个函数在平时使用的时候参数是一个字符串，而在http传输的时候切不可使用引号，否则会返回错误。

列目录发现flag！

构造readfile读flag

base_convert(2146934604002,10,36)(hex2bin(dechex(47)).base_convert(25254448,10,36))

WEB-INF

WEB-INF是Java的WEB应用的安全目录。如果想在页面中直接访问其中的文件，必须通过web.xml文件对要访问的文件进行相应映射才能访问。WEB-INF主要包含一下文件或目录：

/WEB-INF/web.xml：Web应用程序配置文件，描述了 servlet 和其他的应用组件配置及命名规则。

/WEB-INF/classes/：含了站点所有用的 class 文件，包括 servlet class 和非servlet class，他们不能包含在 .jar文件中

/WEB-INF/lib/：存放web应用需要的各种JAR文件，放置仅在这个应用中要求使用的jar文件,如数据库驱动jar文件

/WEB-INF/src/：源码目录，按照包名结构放置各个java文件

/WEB-INF/database.properties：数据库配置文件

下载该文件，根据其文件的分类规则我们推出其文件的位置
WEB-INF/classes/com/wm/ctf/FlagController.class

解码得到flag。
